History Tab

The History tab shows the repository of all check-ins (scripts), whether done by you or other project members, in the order in which they were applied to the project.

Each script listing shows additional details including who made the change, when it was made, and the state of the script.

Script States

In order to understand various script states, here's the glossary of terms used in the "State" column of the History display

  • Confirmed: All project members accepted this script
  • Branch: The script was in conflict with a higher priority script and was branched (see Branch)
  • Tentative: Your own script waiting for acknowledgments from other project members
  • Unconfirmed: Somebody else's script waiting for final acknowledgment from its sender
  • Overdue: Your own script that has been tentative for more than two weeks

In many case a script becomes overdue because some project members stopped responding to scripts. You should notify the administrator of the project about such "dead souls." The administrator should either remove those members from the project or change their status to Observer.

Details Pane

The Details pane at the bottom shows all the files affected by a particular script. From within this pane you can:

  • Double-click a file to view the changes in the visual differ
  • Save selected files either before or after the script to a designated location
  • Select a file and add it to the history filter and later remove it

History Tab Functions

It is from within the History tab that you can do any of the following:

  • Review historical versions
  • Merge branched scripts that were the result of a conflict
  • Branch your project
  • Restore previous version of specific files
  • Restore previous project version
  • Export historical project version
  • Comparing versions in history
  • Filtering project history
