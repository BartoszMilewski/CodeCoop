How do I reconfigure our Hubs to distribute scripts over the LAN?


If you have multiple groups on your LAN with their own Satellite/Hub configurations, you can configure the Hubs to collaborate over the network instead of through Email. Keep in mind, this is for LAN only collaboration. If you have a Hub which also has off-site project members, you will not want to changes your current settings.

  1. Check the configuration of all Hubs. Double-click the dispatcher icon in your task tray. This will open "Collaboration Settings" (Network Hub should be selected)
    • Make sure "Use email" is NOT checked. You won't be using email
    • Below "This Hub's email address (or name)" click "Advanced". Select Alternative Transport tab. Do NOT change the Hub's name (this is usually its email address or fake email). Change "Hub transport": enter the share name of this Hub's public Inbox. For instance, if your Hub machine is called DENVER, type in \\DENVER\CODECOOP .
  2. When another person/workstation wants to join the project, send them an invitation (Project > Invite). It will contain the Hub name and transport which they will need to join the project.
  3. The joiner will do the following:
    • Select Project > Join
    • In the frame "Who invited you to this project?" select "Somebody from a different Hub or peer". Click the "Advanced" button and enter the name of the target Hub (the Hubs name or email address) and then type in that Hub's share name. For instance if you are joining from PORTLAND to DENVER, type in \\DENVER\CODECOOP
